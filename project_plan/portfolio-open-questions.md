# 作品集（portfolio）需求待澄清清单

> 创建日期：2026-04-20
> 用途：在正式进入 `/requirements-discuss` 之前，把 AI 梳理出的所有"未定项"列齐；用户可直接在这上面批注，或把它当成 /requirements-discuss 的议程表逐条过一遍，减少多轮问答的往返。
> 作品集定位：**独立于现有 `/projects`（开源仓库）的作品集实体**，用于展示非代码成品、设计作品、可视化 demo、往期写作合集等。

## 2. 条目字段 (frontmatter)

必填 / 选填请标注。建议对齐现有 project / doc 的约定减少心智负担。

| # | 字段 | 类型 | 必填？ | 说明 |
|---|------|------|--------|------|
| 2.1 | `title` | string | 必填 | 作品名 |
| 2.2 | `slug` | string | 必填 | URL 段；是否自动从 title 推导 |
| 2.3 | `cover` | image path | 选填 | 封面图；本地路径（`/images/...`）or 外链 or 两种都接受 |
| 2.4 | `description` | string(短) | 选填 | 列表页一句话描述（80 字以内） |
| 2.5 | `body` | Markdown | 选填 | 详情页正文，支持 goldmark 全套 + KaTeX |
| 2.6 | `demo_url` | url | 无 | 可选：演示/外链地址 |
| 2.7 | `source_url` | url | 无 | 可选：源码仓库链接 |
| 2.8 | `category` | string | 选填 | 是否按类别分组（设计 / 可视化 / 写作合集 / ...） |
| 2.9 | `tags` | [string] | 选填 | 是否共享 docs 的 tag 云（合流到 `/docs?view=tag`）：独立tag |
| 2.10 | `order` | int | 选填 | 手动排序权重；否则按 updated 倒序 |
| 2.11 | `created` / `updated` | date | 自动填充 | 同 docs |
| 2.12 | `status` | draft/published/archived | 自动填充 | 见 1.2 |
| 2.13 | `featured` | bool | 自动填充 | 见 1.3 |

**关键抉择**：
- **封面图必选还是可选？** 必选 → 列表页视觉整齐；可选 → 部分纯文字作品也能收录。建议必选，无图的作品归到 docs 更合适
  - 必选
- **body 必要性？** 如果主要场景是"列表卡片 + 点外链"，body 可选；如果想有详情页讲背景故事，必填
  - 必选

## 建议的 /requirements-discuss 起点

走 `/requirements-discuss` 时，先一次问 3 个最核心的，其余在对话中自然展开：

1. **封面图必选还是可选？**（决定 2.3、3.2 的布局）
   - 可选，没有用默认图片
2. **有没有"作品详情页 body" 的需求？**（决定 2.5、3.3、编辑器复杂度）
   - 有
3. **是否需要主页仅展示 featured 的若干条？**（决定 1.3、3.7）
   - 需要

这三个答案定下来，其它细节基本能推导。
