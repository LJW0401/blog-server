# 网站构建想法

## 整体设计风格
- 参考 Apple 官网：克制的极简、大留白、克制的色彩
- 字体栈：`-apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text", "PingFang SC"`
- 主色板：
  - 背景：`#fbfbfd` → `#f5f5f7` 渐变
  - 文字主色：`#1d1d1f`，次级 `#6e6e73`，弱化 `#86868b`
  - 强调色：`#0071e3`（Apple Blue），悬浮态 `#0077ed`
- 圆角：大卡 18px、中卡 12px、按钮 980px（胶囊）
- 缓动：`cubic-bezier(0.28, 0.11, 0.32, 1)`

### 背景氛围
- 每一页铺设多颗大尺寸柔光光斑（radial-gradient + `blur(80px)` + `mix-blend-mode: multiply`）
- 配色：蓝、粉、绿、紫、橙、青（淡彩），错峰排布
- 动效：
  - `breathe` 呼吸动画：`scale 1 → 1.12`、`opacity 0.32 → 0.72`，周期 9–14s
  - `drift` 漂移动画：缓慢位移，周期 20–30s，每颗独立错峰
- 尊重 `prefers-reduced-motion`，降级为静态低透明度

## 页面类型

### 主页
- 通过域名可直接访问到的页面
- **吸附滚动（CSS scroll-snap mandatory）**：
  - 第 1 页为完整视口页面，不可停留在中间
  - 下拉过半 → 吸附到第 2 页
  - 上拉过半 → 吸附回第 1 页
  - 继续下拉 → 吸附到底部信息区
- 实现：`html { scroll-snap-type: y mandatory }`，每个 section 设 `scroll-snap-align: start; scroll-snap-stop: always`

#### 顶部导航
- 固定位置、毛玻璃半透明（`backdrop-filter: blur(20px)`）
- 左侧 Logo / 品牌名，右侧导航链接（首页、项目、文档、联系）
- 小屏自动隐藏链接

#### 第 1 页：两栏布局
| 左栏 | 右栏 |
|-|-|
| 显示基本信息 | 最近在活跃的个人仓库 |

**左栏（基本信息）**
- 大字号标题（渐变灰文字 `linear-gradient(180deg, #1d1d1f → #434344)` + `background-clip: text`）
- 次级 tagline，描述个人定位
- 元信息列表：坐标 / 方向 / 现状
- CTA 按钮组：主按钮（胶囊填充）+ 次按钮（文字链接）

**右栏（活跃仓库）**
- 小号 UPPERCASE 栏目标签（如 "Recently Active"）
- 仓库卡片列表，每张卡片包含：
  - 仓库名 + 相对时间（今天 / 2 天前 / 上周）
  - 简短描述
  - 语言色点 + commit 数
- Hover 上浮 2px + 柔和阴影

**页底滚动提示**：底部中央的 "Scroll ↓" 字样，带上下呼吸动画

#### 第 2 页：三行块

**第 1 行块：个人详细信息**
- 栏目标题 "关于我。" + 右侧 "联系 ›" 跳转
- 顶部一段大字号 bio 段落（`~20px`），限制最大宽度 820px，混排强调文本与弱化文本
- 下方 3 列信息卡：
  - **技能栈（Stack）**：pill 胶囊标签集合（TypeScript / Node.js / Go / PostgreSQL 等）
  - **经历（Experience）**：左右布局的时间线列表，右侧年份用 `tabular-nums` 对齐
  - **兴趣（Interests）**：纯条目列表
- 卡片统一：白底、细边、18px 圆角、hover 轻微上浮 + 柔和阴影
- 每张卡片头部：UPPERCASE kicker + 中文副标题

**第 2 行块：主要开源项目**
- 栏目标题 + "查看全部 ›"
- 3 张方形大卡（1 : 1.1 比例）
- 风格混排：深色渐变款（`#1d1d1f → #2d2d30`）、浅蓝渐变款、素色款
- 卡片内容：小号 kicker（分类）+ 大号项目名 + 简述 + 圆形箭头按钮
- Hover：上浮 4px + 右上径向高光淡入 + 箭头右移

**第 3 行块：主要个人总结文档**
- 两列列表
- 每项：文章标题 + meta（分类 · 阅读时长）+ 右侧 `›` 箭头
- Hover：背景加深 + 右移 + 箭头变强调色

### 管理页面
- 通过特殊的路径，如 `domain/manage` 访问
- 进入页面需要密码验证保证安全性（不能用 js，可以被拦截）
  - 倾向：服务端 Basic Auth 或服务端渲染的登录表单，校验在服务端完成
  - 避免把校验逻辑放到前端

## 交互细节
- **入场动画**：`.reveal` 类元素通过 `IntersectionObserver` 在进入视口时触发 `translateY(14px) → 0` + 透明度 0 → 1，相邻元素间错开 60ms 延迟
- **按钮悬浮**：主按钮微微上移 0.5px，过渡 220ms
- **卡片悬浮**：统一使用 `translateY` + 阴影，过渡 300–400ms
- **响应式**：≤860px 两栏折叠为单栏，项目卡与文档列表退化为单列

## 技术栈取向
- 当前原型：纯 HTML + CSS + 少量 JS，无框架依赖
- 后续可能演进方向（待定）：
  - 服务端：用于管理页鉴权、数据接口
  - 数据源：GitHub API 拉取"最近活跃仓库"，本地/CMS 管理开源项目与文档列表
